What are layout views in ASP.NET Core

It is a web page that is responsible for containg presentation logic template.

In ASP.NET Core, layout views refer to the Razor views that define the overall structure and design of a web application’s user interface. Layout views provide a consistent layout across multiple pages and can include common elements such as headers, footers, navigation menus, and other UI components.

A layout view is typically defined in a file with the extension “.cshtml” and contains HTML markup with Razor syntax. The Razor syntax allows for dynamic content to be inserted into the layout view, such as page-specific content, data from a database, or content from a view component.

Layout views can be used with the Razor Pages or MVC (Model-View-Controller) frameworks in ASP.NET Core, and can be customized to meet the specific needs of a web application. By using layout views, developers can save time and effort by reusing common UI elements across multiple pages and maintaining a consistent look and feel throughout the application.

How do the views work together

There are 4 main types of views:
1. _ViewImports.cshtml: it is used to import namespaces, directives, and other settings that apply to all views withi a specifc directory or application.
2. _ViewStart.cshtml: it works like _ViewImports.cshtml, but it can override other setting files that are situated on top of it. As you can see at image, the file in the folder can override the setting of the _ViewStart.cshtml file that is out of the Home folder.


3. View.cshtml: it is basic html + c# file, it’s used to render the page.
4. LayoutView.cshtml: we can use this file to rende others views into it by using @RenderBody() method to import other views to render at the place. And we also have to define the layout file in the view which we want to render, by the way we can set it in the _ViewStart file to provide the setting to all the views in the folder, by using @{Layout = “~/Views/Shared/_Layout.cshtml”;}.
