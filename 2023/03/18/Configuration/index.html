What is the configuration in a ASP.NET Core application

Configuration or configuration settings are the constant key value pairs that are set at a common location and can be read from anywhere in the same application. We can fetch or store the information with it, there are 5 common place to do it:
1. appsettings.json
2. Environment Variables
3. File Configuration(JSON,INI or XML files)
4. In-Memory Configuration
5. Secret Manager

launchSettings.json and appsettings.json

We can set the environment to choose the setting file, if we set it for development then when launch the application the appsettings.Development.json will be choosed

IConfiguration

We can use the IConfiguration to fetch datas from the configuration file

Environment Specific Configuration

The precedence of configuration sources by low to high is that: appsettings.json -> appsettings.EnvironmentName.json -> Secret Manager -> Environment Variables -> Command Line Arguments
It is better to create a model to provide those configurations, because we won’t to write a lot of keys manualy

Secrets Manager

It can store the user secrets, some sensitive configuration data in a separate location on the developer machine. Because we don’t want to show up our sensitive data publicly.
1. Open the terminal on project level
2. Initialize user secrets, it will give you a user secret id: dotnet user-secrets init
3. Set the key value pair, it will be stored in the usersecrets: dotnet user-secrets set “key” “value”
4. We can find out that secrets by right click the project -> manage user secrets
5. The latest configuration will override to the previous one, it means by using secrets manager will override the setting of the appsettings.json if there are some same keys
The benifit of the usersecrets is that it will avoid store the information in the source controller, because it is not safe to store the sensitive information in the source code.

Environment variable configuration

We can create configuration in the environment: $Env:key__childkey=”value”
