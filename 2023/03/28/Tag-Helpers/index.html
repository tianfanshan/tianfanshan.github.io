What are Tag Helpers

Tag helpers in ASP.NET Core are a type of server-side code that allows developers to create HTML elements that are associated with server-side behaviors. They provide a way to create HTML markup using C# or other programming languages, making it easier to create dynamic web pages and web applications.
Tag helpers work by mapping HTML tags to server-side code, which can perform a variety of actions. For example, a tag helper might be used to display data from a database or to generate a form field that is associated with a particular model property.
ASP.NET Core includes several built-in tag helpers for common scenarios, such as form elements, links, and images. Developers can also create their own custom tag helpers for more specialized scenarios.
Tag helpers provide several advantages over traditional server-side code approaches, such as server controls and Razor syntax. They allow developers to write HTML markup in a more natural way, while still providing the benefits of server-side processing and code reuse.

Tag Helpers

  1. for “input”, “textarea” and “label”: asp-for
  2. for “a” and “form”: asp-controller, asp-action, asp-route-x, asp-route and asp-area
  3. for “select”: asp-for and asp-items
  4. for “img”: asp-append-version
  5. for “span”: asp-validation-for
  6. for “script”: asp-fallback-src, asp-fallback-test
  7. for “div”: asp-validation-summary

Binding: The form tags such as “input”, “label”, “textarea” and “select” can be bound with specific model properties. It applies model property name to name and id attributes of “input” tag.
Url Generation: Route URL will be re-generated for “a” and “form” tags. It generates the url as controller/action pattern.
