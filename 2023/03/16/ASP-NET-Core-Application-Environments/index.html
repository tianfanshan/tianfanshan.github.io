ASP.NET Core Application Environments

2023-03-16

Environments

ASP.NET Core supports multiple environments, which are different runtime contexts that your application can run in. Each environment typically has its own set of configuration settings, allowing you to customize your application’s behavior based on the current environment.

The most common environments in ASP.NET Core are:
1. Development: This is the default environment when you’re running your application locally. In this environment, features such as detailed error messages and auto-reloading of the application are enabled to help with development.
2. Staging: This environment is used for testing your application before deploying it to production. In this environment, you can simulate the production environment by using production-like configurations and test your application against realistic scenarios.
3. Production: This is the environment where your application is deployed and used by end-users. In this environment, you typically want to optimize performance and minimize the risk of errors or security vulnerabilities.

Also depends the environment we can write some logic code in our controller by using those method or filed of the IWebHostEnvironment object: IsDevelopment(), IsStaging(), IsProduction(), IsEnvironment(string environmentName), ContentRootPath and EnvironmentName.

Terminal

We can run the application on terminal by using the command dotnet run.


It is possible to run the application without the setting profile by using the command dotnet run –no-launch-profile, but it by default will run on the Production environment.

We also can change the environment manually by using the commands: $Env:ASPNETCORE_ENVIRONMENT=”Development” or set ASPNETCORE_ENVIRONMENT=”Staging”.
