What is Migration in ASP.NET Core

Migration in ASP.NET Core refers to the process of updating a database schema to match changes in the application’s data model.
In ASP.NET Core, migrations are managed using the Entity Framework Core (EF Core) tool. EF Core allows developers to define a database schema using C# code, and then automatically generate SQL scripts to create or update the database schema based on changes to the code.
To use migrations in ASP.NET Core, you first need to create a data context class that inherits from the DbContext class in EF Core. Then, you define entity classes that represent the tables in your database, and add them to the data context using DbSet properties.
Once you have defined your data model, you can create a migration by running the dotnet ef migrations add command in the command line. This will generate a new migration file that contains the SQL statements necessary to update the database schema.
Finally, you can apply the migration to the database by running the dotnet ef database update command. This will execute the SQL scripts generated by the migration file, and update the database schema to match the data model defined in your code.

  1. Install Microsoft.EntityFrameworkCore.SqlServer in start up project, and also Microsoft.EntityFrameworkCore.Design.
  2. Install Microsoft.EntityFrameworkCore.SqlServer in entity project, and also Microsoft.EntityFrameworkCore.Tools.
  3. Creates Migration by using the command on the Pachage Manager Console in the entities project: Add-Migration MigrationName.
  4. Updates or executes the migration also by using the command on the Pachage Manager Console in the entities project: Update-Database -Verbose
