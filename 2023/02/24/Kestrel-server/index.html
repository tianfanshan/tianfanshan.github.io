Kestrel server

2023-02-24

What is kestrel server

It is the default cross platform http server for ASP.NET core application, but we only use it as a development server. Once we complited the application, it requires to push the code into the production server, for example: 

1. IIS
2. NGINX or apache

Because kestrel doesn’t support some of advanced server feature, that are required on internet in these days, we can use Reverse Proxy Server to complite those features: 

1. load balancing
2. URL rewriting
3. Authentication
4. caching
5. Decompressing the request
6. Decryption of SSL certificates

Once we want to deploy our application on the Reverse Proxy Server, it will transfer the request to kestrel. At moment we can use IIS Express to simulate IIS.

Without Reverse Proxy Server works like:

Internet <--HTTP--> kestrel <----> Application code

With Reverse Proxy Server works like:

Internet <--HTTP--> IIS/Apache/NGNIX <----> kestrel <----> Application code

When we run the application there the terminal will pop out, and you can see the messages of the kestrel server show on it

The launchSettings.json file

We can configure the properties of the server in this file:


1. dotnetRunMessages: true; to show the messages of server on the terminal
2. launchBrowser: true; it will automatically open the browser
3. applicationUrl: url; to set the root url

1. windowsAuthentication: false; enable the windows authentication or not
2. iisExpress.applicationUrl: url; to set the IIS Express root url
