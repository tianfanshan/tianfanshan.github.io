Spring Boot Transactional Demo

2023-02-14

Demos of @Transactional annotation

  • In real-world application like e-commerce, when the client solicited a order with wrong payment, there is necessary to use @Transactional to rollback.
    Because it doesn’t make sense to keep all the information in the database if payment is not available.

    • At first create a simple Springboot REST api, is just a normal rest api, nothing special.

    • Then in the service implementation the workflow is: Save order details -> Save payment details. If we don’t add @Transactional annotation even if the payment type is wrong it will be stored in our database the order without payment.

    • By adding @Transactional annotation, if payment failed for this order, then it won’t save in database, and the saved order will get rool back.

    • We don’t have to use @EnableTransactionManagement in the main class, because whenever we use @Transactional annotation in SpringBoot Application the Spting boot by default enable transaction management.
