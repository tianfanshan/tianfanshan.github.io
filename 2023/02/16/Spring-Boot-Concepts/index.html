Spring Boot Concepts

2023-02-16

What is Spring Boot

Spring boot is basically an extension of Spring framework, it is builted on top of spring framework to reduce a lot of configuration required to set up the spring based applications.
Spring boot allows the developer to quickly create production Spring-based applications without requiring developers to werite the same boilerplate configuration again and again.

A few important Spring Boot features

Spring Boot Starters
It offers many starter models to getting starter quickly: Spring MVC, JPA, MongoDB, Spring Security etc…
We no longer have to search for compatibility library version and configure them manually.

Spring Boot Autoconfiguration
It attempts to automatically configure a Spring Application based on the jar dependencies that you have added.
Without is we have to configure a lot things:
When we use Spring MVC: Component Scan, Dispatcher Servlet, View resolver and Web jar.
When we use Hibernate/JPA: Dtasource, Entity manager factory/session factory and transaction manager amon a host of other things.

Externalized Configuration
It enables developer to work with the same application in different environment: dev, prod, qa etc…
It allows us to use properties file to mantain the configuration.

Spring Boot Actuator
Spring Boot provides a wide variety of production ready features without requiring developers to write a much code, we can use it to view:
The application bean configuration details
The application URL mappings
The environment details
Configuration parameter values
The registered health check metrics

Easy-to-use embedded servlet container support
We can quickly run our Spring application in a better servlet container, because it by default provides Tomecat server as a default servlet container, so we no longer haver configure the Tomcat server.

How Spring Boot internally works

The workflow is:

  1. Spring Boot Application execution will start from main() method
  2. The main() method internally call SpringApplication.run() method
  3. SpringApplication.run() method performs bootstraping to load all the necessary configuration and components required to run over Spring Boot Application
  4. Spring Boot starts StopWatch to calculate the time taken by our Spring Boot Application for bootstraping data
  5. Prepare environment to run our Spring Boot Application by using the properties file
  6. Print the banne on the console(Spring Boot logo)
  7. Spring Boot starts the IOC container based on the classpath, if we don’t use Spring Boot we have to manually start the IOC container
  8. Refresh context, it will load all the necessary components
  9. Trigger Runners
  10. run() method returns application context object(IOC)

Different ways to create Spring Boot Application

  1. Using Spring initializr
  2. Using Spring Starter Project in STS
  3. Using Spring Boot CLI

@SpringBootApplication, @Configuration and @ComponentScan

The @SpringBootApplication is basically the combination of @SpringBootConfiguration, @ComponentScan and @EnableAutoConfiguration. And @SpringBootConfiguration internally uses @Configuration annotation.
We can make a Java class a configuration class by adding @Configuration on it. Then we can define @Bean inside this class to configure our project.
The Spring Boot Application is using @EnableAutoconfiguration to enable auto configuration classes, whenever we use spring boot application it will find dependency on a class path.
We use @ComponentScan u scan components in a particular package.

What is Spring Boot starters

It basically provides a starter dependency like : spring-boot-starter-web. And it is internally managing a lot of dependencies to solve the compability problem. Because before we use Spring Boot to create a project we have to manually configure the version of each dependency.

What is Spring Boot starter parent

It provides the default properties internally: version of Java, a lot of plugins that required for Maven project, the version of the all dependencies etc…

How many types of projects we can create using Spring Boot

There are 3 types of project we can create with Spring Boot by using:

  • spring-boot-starter, it can create utilities, stand alone and desktop based(GUI) projects.
  • spring-boot-starter-web, it can create spring MVC web application and spring MVC REST API application.
  • spring-boot-starter-webflux, it can create Spring Boot Reactive Web application.

Types of Embedded Servers in Spring Boot

The reasion why do we need a embedded server in the application, because if we don’t use it then it’s necessary to download and set up the server and also have to deploy the application manually.
By default the dependency spring-boot-starter-web internally provides Tomcat server, and there are two more embedded services: Jetty and Undertow, we can use in our project instead of using Tomcat. To use another embedded server we have to exclude the Tomcat server, at first add the exclusions tag in spring-boot-starter-web, and excluding the default Tomcat server, then add new server dependency in the pom file.

How to run Spring Boot application from command line

We have two options for running it: using java -jar or using Spring Boot Maven plugin -mvn spring-boot:run.
When we run the application without installed Maven in our machine the the mvnw.cmd will install Maven for us to run the application.

  1. Open terminal in the project folder
  2. Package Spring Boot Application as jar file by using command: mvnw package
  3. Run application if Maven installed: mvn spring-boot:run
  4. Run application if Maven is not installed: mvnw spring-boot:run
  5. Or go to the target level by using command: cd target
  6. Run application: java -jar fileName-SNAPSHOT.jar

Describe the flow of REST API HTTP request through the Spring Boot project

There are 3 important layers: Controller, Service and DAO
Controller layer, it is responsible to process the requests, it’s also called API layer.
Service layer, it is used to mantain all the business logic, if we have any third part REST API call or any public third part REST API call, we can keep them in a service layer.
DAO or repository layer, it’s responsible to talk to the database, it holds all the database logic or persistence logic.
The flow: client -> controller -> service -> DAO -> database
The flow: database -> DAO -> service -> controller -> client

How to create and deploy Spring Boot WAR to External Tomcat Server

When we create a spring boot war project, it is necessary to choose war the packaging file type instead of jar. Once we created the war project we can se there are 3 difference: in pom.xml there is packagin with war, a tomcat dependency with scope provided and a ServletInitializer class in project.
How to run it to External Tomcat Server:

  1. Build Maven
  2. Download Tomcat to our machine
  3. copy paste war file from target that created by using Maven builder to webapp dir of tomcat folder
  4. click startup icon of bin folder of tomcat
  5. Use war file name as RUL address behind port

What is Spring Boot DevTools used for

Without added spring-boot-starter-devtools dependency, we have to restart the application manually when a developer changed the code to show it up on the application.
Once we added spring-boot-devtools in our application, spring boot will detect our changes in the project, and based on that , it will restart the server automatically.

How to add security to Spring Boot and Explain Security Auto Configuration

Add spring-boot-starter-security in pom.xml, it will add all this spring boot related libraries and the compatible version. When we run the application, spring boot will provides a random password for us on the console, then we can use it with the default userid user to log-in our application to get the permission to use the application.
We can also customize our username and password by configuring the properties file:

  • spring.security.usernam=?
  • spring.security.password=?
  • spring.security.roles=?

Use profiles in Spring Boot project

It means we can create different properties file to use for specific environment, and we can active the specific properties file by using: spring.profile.active=?
There are 3 examples we were used in our application:

  • application-prod.yml
  • application-dev.yml
  • application-qa.yml
