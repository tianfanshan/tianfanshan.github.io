Why do we use The Repository pattern in our project

It is commonly used in .NET Framework projects to provide a separation between the business logic of an application and the data access layer.
By using the Repository pattern, the application’s data access logic is encapsulated within a separate class or layer, which allows for easy maintenance and testing of the code. This also helps to promote a more modular architecture, where individual components can be swapped in and out without affecting the rest of the system.
In addition, the Repository pattern can help to improve the scalability and performance of a .NET Framework project by allowing the data access layer to be oprimized independently of the rest of the application. This can include techniques such as caching, indexing, and partitioning of data.

Implement the Repository pattern in our project

  1. Create interface project to connect the services and the repositories
  2. Create the repositories project to access the database
  3. Instead of using ApplicationDBContext in the services we use the interface of those repositories to access to the database
  4. And also it is necessary to add those interfaces into the Service to make it work
